# 共同編集の手順

## なぜ GitHub でテストが失敗しているか

- いま **ローカル** ではテスト修正済みで全テストが通っています。
- **GitHub 上** には、その修正がまだ送られていません（古いコードのまま）。
- CI は「GitHub にあるコード」でテストするので、**push するまで GitHub のテストは失敗したまま**です。

**やること**: 修正済みのコードを push する → GitHub 上でも同じテストが走り、通るようになります。

---

## 手順 1: いまの修正を GitHub に push する

### 1-1. 送るファイルを選んでコミットする

**送るもの**: テスト・コード・設定の修正。**送らないもの**: 本番データ（admins.json など）や coverage レポート。

PowerShell でプロジェクトフォルダに移動してから、次を実行します。

```powershell
cd c:\Users\user\Desktop\program\web-order

# 設定・コード・テスト・ドキュメントだけ add（データファイルは含めない）
git add .gitignore .cursorrules
git add jest.config.js playwright.config.js
git add dbPaths.js
git add routes/admin-api.js services/mailService.js services/stockService.js
git add tests/
git add docs/
git add scripts/run-test-all.js

# コミット（メッセージは変更内容に合わせて書き換えてよい）
git commit -m "テスト修正: CI/E2E用サンドボックス・代理ログイン・excel-reader・captchaリセット先"
```

※ `tests/` を add するとテスト関連はまとめて入ります。`tests/_sandbox_data/` は .gitignore に入れたので add されません。

**意図的に add していないもの**（本番データ・生成物なので push しない）:
- `admins.json`, `customers.json`, `orders.json`, `products.json`, `stocks.json`, `support_tickets.json` など
- `coverage/` フォルダ
- `upload.xlsx`, `kaitori_master.json`, `kaitori_requests.json`, `config/stocks-adapters.json`（ローカル環境用）

### 1-2. push する

```powershell
git push origin main
```

※ ブランチが `master` の場合は `git push origin master` にしてください。

### 1-3. GitHub で CI の結果を確認する

1. ブラウザで https://github.com/hfmmmi/web-order-co を開く
2. **Actions** タブをクリック
3. いちばん上のワークフロー（直近の push）を開く
4. **test** ジョブが緑（✓）になっていれば、GitHub 上のテストも通過しています

## 手順 2: （任意）マージ前にテスト必須にする

共同編集で「テストが通っていない変更はマージできない」ようにしたい場合:

1. リポジトリの **Settings** → **Branches**
2. **Add branch protection rule**
3. **Branch name pattern**: `main`（または `master`）
4. **Require status checks to pass before merging** にチェック
5. 一覧から **test** を選択
6. 保存

くわしくは `docs/ci-setup.md` を参照してください。

## 手順 3: 共同編集者側の流れ

相手方の作業イメージです。

1. **リポジトリをクローン**
   ```powershell
   git clone https://github.com/hfmmmi/web-order-co.git
   cd web-order-co
   npm install
   ```

2. **作業用ブランチを作って編集**
   ```powershell
   git checkout -b feature/○○
   # ファイルを編集
   ```

3. **テストをローカルで実行**
   ```powershell
   $env:DATA_DIR = "tests/_sandbox_data"
   npm run test:all
   ```

4. **コミットして push**
   ```powershell
   git add （変更したファイル）
   git commit -m "○○の対応"
   git push origin feature/○○
   ```

5. **GitHub で Pull Request（PR）を作成**
   - ブランチ `feature/○○` から `main` への PR を作成
   - CI が自動で走るので、緑になってからマージ

6. **あなた（オーナー）が PR をマージ**
   - 内容を確認して **Merge pull request** で取り込み

---

## まとめ

| 順番 | 誰が | やること |
|------|------|----------|
| 1 | あなた | 修正を push → GitHub のテストを緑にする |
| 2 | あなた | （任意）ブランチ保護で「テスト必須」を有効化 |
| 3 | 共同編集者 | クローン → ブランチで編集 → テスト → push → PR 作成 |
| 4 | あなた | PR を確認してマージ |

まずは **手順 1** の push を実行すると、GitHub 上のテストが通るようになります。
