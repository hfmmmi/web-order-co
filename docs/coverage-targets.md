# カバレッジ閾値（目標）

## 目標値

| 対象 | 行カバレッジ | 分岐カバレッジ |
|------|-------------|---------------|
| **全体（最低限）** | 60% | 45% |
| **全体（理想）** | 80% | 70% |
| **P0（重要）** | 80% | 70% |

## P0 対象（必須で 80/70 を達成する）

- `routes/auth-api.js` … 認証・失敗ロック・CAPTCHA
- `routes/orders-api.js` … 注文確定
- `routes/admin-api.js` … 主要設定更新
- `middlewares/validate.js` … 入力検証統一
- `utils/priceCalc.js` … 価格計算

## 現在の設定

`jest.config.js` の `coverageThreshold` はテスト追加で引き上げ済みです。

| 対象 | 現在の閾値 | 実際値（2026-02-17） | 目標 |
|------|-----------|----------------------|------|
| global | 行42%・分岐24% | 行60.6%・分岐37.88% | 行60%・分岐45% |
| auth-api.js | 行75%・分岐60% | 77%・63% | 行80%・分岐70% |
| orders-api.js | 行90%・分岐62% | **91%・63%** | 行80%・分岐70% ✓ |
| admin-api.js | 行75%・分岐56% | **76%・58%** | 行80%・分岐70% |
| validate.js / priceCalc.js | 達成済み | 95%・87% / 100% | 行80%・分岐70% ✓ |

目標を満たすにはさらなるテストの追加が必要です。

### 段階的な引き上げ

1. `npm run test:api` でカバレッジを確認
2. 未カバーの行・分岐にテストを追加
3. `jest.config.js` の閾値を少しずつ引き上げて再実行
