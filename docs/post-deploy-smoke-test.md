# デプロイ後スモークテスト（手順）

本番またはステージングに反映した**直後**に実施する、短い手動確認です。自動テストを**代替**しません（併用）。

環境: HTTPS でブラウザからアクセスできること（[production-environment-checklist.md](production-environment-checklist.md)）。

---

## 手順

各項目で **期待どおり**ならチェック。失敗時はロールバックまたは修正を検討。

### 顧客側

- [ ] トップまたは商品一覧が表示される（重大な JS エラーがない）。
- [ ] ログインができる（テスト用アカウントで可）。
- [ ] ログイン後、価格・カート等、主要画面が表示される。
- [ ] **注文フロー**: テスト注文を1件入れ、確定まで到達できる（本番で危険ならステージングのみ実施し、本番は閲覧＋カートまで等に落とす）。
- [ ] 注文通知メール・顧客向けメールが仕様どおり届く（テスト宛先を使う）。

### 管理側

- [ ] 管理ログイン URL にアクセスし、管理者でログインできる。
- [ ] 受注一覧またはダッシュボードが開ける。
- [ ] （利用中なら）商品・顧客・設定のいずれか1画面を開き、保存不要の読み込みが成功する。

### 共通

- [ ] ブラウザの開発者ツールの **Console** に、操作に応じて致命的な赤エラーが連発していない。
