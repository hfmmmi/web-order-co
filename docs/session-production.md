# 本番環境でのセッション運用（再起動後もログイン維持）

運用中にサーバーを再起動（デプロイ・更新・メンテナンス）しても、**ユーザーに再ログインさせない**ための設定です。

## 1. 再起動後もログインを維持する（推奨）

セッションはファイルで保存されています。**保存先を「デプロイで消えないディレクトリ」にすると**、再起動後も同じファイルを読みにいくため、ログイン状態が維持されます。

### 設定方法

環境変数 **`SESSION_PATH`** に、アプリのデプロイ先とは別の永続ディレクトリを指定してください。

| 環境の例 | 設定例 |
|----------|--------|
| 自社サーバー（Linux） | `SESSION_PATH=/var/lib/weborder/sessions` |
| Docker（ボリュームをマウント） | `SESSION_PATH=/data/sessions`（/data をボリュームに） |
| PM2 などで常駐 | アプリディレクトリ外のパスを指定 |

- 指定したディレクトリは**自動作成されません**。初回デプロイ前に `mkdir` で作成してください。
- アプリを更新してコードを上書きしても、`SESSION_PATH` で指したフォルダは触らないようにしてください。

### 確認

- 起動ログに「SESSION_PATH を永続ディレクトリに設定してください」と出ている場合は、上記が未設定です。
- 設定後は再起動しても、同じブラウザではログインしたまま操作できます。

## 2. やむを得ずログアウトした場合のメッセージ

次のような場合は、セッションが無効になり再ログインが必要になります。

- **120分間** 何も操作しなかった（自動ログアウト）
- サーバーを**別マシンに移した**／セッション保存先を変えた
- **SESSION_PATH を未設定**のままデプロイし、再起動で `sessions` フォルダが消えた

その場合、画面には  
「再ログインが必要です。（長時間の無操作やシステム更新によりセッションが切れた場合があります）」  
と表示され、ログイン画面に誘導されます。ユーザーには「不具合」ではなく「仕様」であることが伝わるようになっています。

## 3. まとめ

| 目的 | 対応 |
|------|------|
| 再起動・デプロイ後もログイン維持 | 本番で `SESSION_PATH` を永続ディレクトリに設定する |
| 再ログイン時の不満を減らす | 上記メッセージで理由を案内（実装済み） |
