関連: [operational-risk-index.md](operational-risk-index.md)（全体の目次）

## 1. 「バグ」「ダウン」「迷惑」の分け方

| 種類 | 意味（顧客から見た目） | 例 |
|------|------------------------|-----|
| **バグ** | 画面は開くが、**結果や表示がおかしい** | 価格が違う、注文が片方だけ残る、ボタンで意図と違う動き |
| **ダウン** | **ページが開かない**、ずっと読み込み、API がエラーで止まる | サーバ停止、証明書エラー、503 |
| **迷惑** | システムは動いているが**体験や業務が損なわれる** | メールが届かない、更新のたびに全員再ログイン、極端な遅さ |

**演習:** 次にトラブルが起きたら、まずこの3つのどれかに分類する。対処の優先度と連絡文面が決めやすくなります。

## 2. 原因カテゴリとの対応（要約）

| 原因 | 典型的な見え方 |
|------|----------------|
| 実装バグ・仕様漏れ | バグ |
| データ破損・競合・誤上書き | バグ または ダウン |
| サーバ・ネットワーク・証明書 | ダウン |
| メール・DNS・CAPTCHA 等 | 迷惑（届かない・ログインできない） |
| 設定ミス・誤操作 | ダウン・データ消失・迷惑のいずれも |

詳細な対策は [production-environment-checklist.md](production-environment-checklist.md)、[cloud-vps-production-setup.md](cloud-vps-production-setup.md)、[backup-and-restore-policy.md](backup-and-restore-policy.md)、[release-test-discipline.md](release-test-discipline.md) を参照してください。

## 3. 顧客ジャーニー例（発注システム向け・3本）

以下は **記入例** です。自社の実フローに合わせて書き換え・追記してください。

### ジャーニー A: 顧客が注文するまで

| ステップ | 止まると何が困るか（例） |
|----------|---------------------------|
| 商品一覧・カート | 注文できない、価格が信頼できない |
| ログイン | 自分の価格ランク・履歴が見えない |
| 注文確定 | 受注が伝わらない、二重注文の恐れ |
| 確認メール | 顧客が証跡を持てない |

### ジャーニー B: 顧客がパスワード再設定する

| ステップ | 止まると何が困るか（例） |
|----------|---------------------------|
| 「忘れた方」から申請 | ログインできず業務停止 |
| メール受信 | リンクが届かず先に進めない |
| setup 画面で新パスワード | 設定できずロックアウトが続く |

### ジャーニー C: 管理者が日次業務する

| ステップ | 止まると何が困るか（例） |
|----------|---------------------------|
| 管理ログイン | 出荷・マスタ更新ができない |
| 受注確認・ステータス更新 | 倉庫・顧客連絡が遅れる |
| CSV/Excel 取込 | 在庫・価格が現場とズレる |
