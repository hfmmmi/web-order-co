# 顧客アドレス（email）消失の原因調査

## 結論

**原因:** E2E テスト実行時に、**プロジェクト直下の本番用 JSON（`customers.json` 等）が E2E 用の初期データで上書きされていた**可能性が高いです。

- **以前「テストですべてのマスタが消えた」**  
  → E2E の `prepare-e2e-db.js` が **プロジェクト直下** をバックアップしたあと、**同じプロジェクト直下** に E2E 用の seed（TEST001 のみの customers など）を上書きしているため。
- **復元後に H015 のアドレスだけない**  
  → 復元に使ったバックアップが「E2E 実行前の古い状態」で、その時点では H015 に `email` を登録していなかった、または別のバックアップ／手動復元で `email` が含まれていなかった可能性があります。

---

## 根拠（コード上の事実）

### 1. E2E が「プロジェクト直下」を読んでいる

- `tests/e2e/scripts/e2eDbManager.js` では  
  `ROOT = path.join(__dirname, "..", "..", "..")`  
  → `__dirname` が `tests/e2e/scripts` なので **プロジェクトルート** を指しています。
- `abs("customers.json")` は **プロジェクト直下の `customers.json`** になります。

### 2. E2E 実行の流れで本番データが上書きされる

- `playwright.config.js` の `webServer.command` で  
  `node tests/e2e/scripts/prepare-e2e-db.js && node server.js`  
  が実行されます。
- `prepare-e2e-db.js` は  
  1. **backup()** … プロジェクト直下の JSON を `.tmp/db-backup.json` に退避  
  2. **seedForE2E()** … プロジェクト直下の `customers.json` 等を **E2E 用の内容で上書き**（顧客は TEST001 のみ、email は `e2e-customer@example.com` のみ）
- つまり **E2E を一度実行するだけで、本番の `customers.json` が E2E 用の 1 件に置き換わります。**

### 3. 復元は「teardown」に依存している

- テスト終了時に `global-teardown.js` の **restore()** で、退避した内容をプロジェクト直下に戻します。
- **teardown が動かなかった場合**（Ctrl+C、プロセス落ち、Playwright の異常終了など）は、**restore が実行されず、本番データが E2E 用のまま残ります。**

### 4. アプリ側の保存処理では email は消さない

- `customerService.js` の `updateCustomer` は  
  `list[index].email = email !== undefined ? String(email || "").trim() : (list[index].email || "");`  
  で、**既存の email を意図的に消す処理はありません。**
- Excel 取込の更新でも `if (inputEmail) customerList[idx].email = inputEmail` で、**空で上書きはしない**ため、通常運用で email だけが消える実装にはなっていません。

したがって、「**E2E の seed でファイルごと差し替え → teardown 未実行 or 古いバックアップで復元**」という流れで、顧客アドレスが消えたと説明できます。

---

## テストをやめる必要はあるか

**いいえ。テストは続けて問題ありません。**  
ただし **「本番データ（プロジェクト直下）を触らない」ようにする**必要があります。

- **API テスト（Jest）**  
  - すでに `tests/helpers/testSandbox.js` で **DATA_DIR 未設定時はエラー** にして、`tests/_sandbox_data` のみ触る運用になっています。  
  - 普段から **`DATA_DIR=tests/_sandbox_data` を付けて** `npm run test:api` を実行していれば、本番の `customers.json` は触りません。
- **E2E（Playwright）**  
  - 上で述べた通り、従来は **プロジェクト直下** を backup/seed/restore していたため、本番データが消える原因になっていました。  
  - 対応として、**E2E 専用のデータディレクトリ（例: `tests/e2e/.e2e_data`）だけを seed し、サーバー起動時に `DATA_DIR` でそのディレクトリを参照する**ように変更すれば、本番データを一切触らずに E2E を実行できます。

上記のように **API テストはサンドボックス、E2E は E2E 専用ディレクトリ** に限定すれば、テストは今後も安全に実行できます。

---

## 再発防止のための対応（実施済み）

- E2E 用に **専用データディレクトリ `tests/e2e/.e2e_data`** を用意し、  
  backup/seed/restore およびサーバー起動時の読み書きを **すべてこのディレクトリに限定**する変更を行いました。
- これにより、**E2E 実行時にはプロジェクト直下の `customers.json` 等は一切触れず**、本番データが消える事象の再発を防ぎます。
